# 00 - Guia de Estudos

Este módulo existe para evitar um erro muito comum: tentar estudar tecnologia sem método. Quem está começando geralmente não trava por falta de inteligência, e sim por falta de organização, repertório e segurança.

## Objetivos do módulo

- Entender como estudar programação sem se perder.
- Preparar o ambiente de trabalho.
- Criar uma rotina simples e sustentável.
- Aprender a tirar dúvidas de forma produtiva.

## Antes de tudo

Se você nunca estudou programação antes, espere sentir:

- estranheza com termos novos
- lentidão no começo
- vontade de comparar seu progresso com o de outras pessoas
- sensação de que está esquecendo tudo

Tudo isso é normal.

## Método de estudo recomendado

Use esta sequência:

1. **Ver**: assistir a uma aula ou ler uma explicação.
2. **Reproduzir**: repetir o exemplo com as próprias mãos.
3. **Explicar**: tentar dizer com suas palavras o que acabou de aprender.
4. **Praticar**: fazer pelo menos um exercício sem copiar.
5. **Revisar**: voltar ao conteúdo depois de um ou dois dias.

## Como fazer anotações úteis

Evite anotações gigantes copiadas da aula. Prefira registrar:

- conceito
- exemplo curto
- dúvida
- erro que aconteceu
- solução encontrada

Modelo simples:

```md
## Tema
- O que é:
- Para que serve:
- Exemplo:
- Dúvida que eu ainda tenho:
- Erro que cometi:
- Como resolvi:
```

## Ferramentas para instalar

- [Visual Studio Code](https://code.visualstudio.com/docs/setup/setup-overview)
- [Git](https://git-scm.com/downloads)
- [GitHub](https://github.com/)
- [Chrome DevTools](https://developer.chrome.com/docs/devtools/open?hl=pt-BR) para inspeção de páginas

## Organização mínima de pastas

Sugestão:

```text
estudos-web/
  anotacoes/
  exercicios/
  projetos/
  referencias/
```

## Como pedir ajuda sem travar

Quando tiver dúvida, tente organizar a mensagem assim:

1. O que eu estou tentando fazer.
2. O que eu esperava que acontecesse.
3. O que aconteceu de fato.
4. O trecho de código ou o passo que causou o erro.
5. O que eu já tentei fazer para resolver.

Esse formato economiza tempo e melhora muito a qualidade da ajuda recebida.

## Regras de convivência com o próprio aprendizado

- Não pule direto para conteúdo avançado porque parece mais "profissional".
- Não trate revisão como perda de tempo.
- Não use tutorial como muleta permanente.
- Não desista de um tema no primeiro desconforto.

## Checklist inicial

- Tenho uma pasta organizada para estudar.
- Instalei um editor de código.
- Tenho um navegador principal.
- Criei uma conta no GitHub.
- Separei um horário fixo de estudo na semana.
- Tenho um lugar para guardar minhas anotações.

## Projeto do módulo

### Projeto: Kit do Estudante de Programação

Monte sua estrutura inicial de estudo:

- criar a pasta `estudos-web`
- criar subpastas para anotações, exercícios e projetos
- instalar VS Code
- instalar Git
- criar conta no GitHub
- salvar nos favoritos os links mais usados da trilha
- criar um arquivo `primeiros-passos.md` com seus objetivos de estudo

## Critério de conclusão

Você pode seguir para o próximo módulo quando:

- souber onde vai estudar
- souber onde vai salvar seus arquivos
- souber quais ferramentas vai usar
- tiver um plano mínimo de rotina
