# Glossário para Iniciantes

## A

- **Algoritmo**: sequência lógica de passos para resolver um problema.
- **API**: forma padronizada de um sistema conversar com outro.
- **Arquivo**: unidade de armazenamento de informação no computador.

## B

- **Backup**: cópia de segurança de arquivos importantes.
- **Browser**: outro nome para navegador.

## C

- **Cliente**: dispositivo ou programa que faz uma requisição a um servidor.
- **Código-fonte**: texto escrito pelo desenvolvedor para criar software.
- **Commit**: registro salvo de alterações em um projeto Git.
- **CSS**: linguagem usada para estilizar páginas web.

## D

- **Deploy**: ato de publicar um projeto para outras pessoas acessarem.
- **DOM**: representação da página HTML como uma árvore que o JavaScript pode manipular.
- **DNS**: sistema que traduz nomes de sites em endereços de rede.

## E

- **Editor de código**: programa usado para escrever código.
- **Elemento HTML**: bloco estrutural de uma página, como título, parágrafo, imagem e botão.
- **Evento**: ação detectada pela página, como clique, digitação ou envio de formulário.

## F

- **Fetch**: recurso do JavaScript para buscar dados de outro lugar, geralmente via internet.
- **Framework**: conjunto de ferramentas e regras para acelerar o desenvolvimento.
- **Função**: bloco de código reutilizável que executa uma tarefa.

## G

- **Git**: sistema de controle de versão.
- **GitHub**: plataforma para hospedar repositórios Git e colaborar em projetos.

## H

- **Hardware**: parte física do computador.
- **Hospedagem**: serviço que mantém um site disponível na internet.
- **HTML**: linguagem de marcação usada para estruturar páginas web.
- **HTTP/HTTPS**: protocolos usados para comunicação entre navegador e servidor.

## I

- **Input**: dado de entrada fornecido por usuário, sistema ou programa.
- **Interface**: parte visual ou ponto de interação de um sistema.

## J

- **JavaScript**: linguagem de programação usada para dar comportamento a páginas web.

## L

- **LocalStorage**: espaço no navegador para guardar dados simples localmente.
- **Loop**: estrutura que repete instruções.

## M

- **Markdown**: forma simples de escrever texto formatado.
- **Media Query**: recurso do CSS para adaptar layout a diferentes tamanhos de tela.

## N

- **Navegador**: programa usado para abrir sites, como Chrome e Firefox.
- **Node**: nó de uma árvore DOM ou ambiente para rodar JavaScript fora do navegador, dependendo do contexto.

## P

- **Pasta**: estrutura usada para organizar arquivos.
- **Push**: envio de commits locais para um repositório remoto.
- **Pull**: atualização do repositório local com mudanças do remoto.

## R

- **Repositório**: pasta controlada pelo Git, com histórico de versões.
- **Responsividade**: capacidade de a interface se adaptar a diferentes telas.

## S

- **Semântica**: uso de elementos com significado correto, como usar `header` para cabeçalho.
- **Servidor**: computador ou serviço que entrega dados, arquivos ou páginas para clientes.
- **Software**: programas e sistemas executados no computador.

## T

- **Tag**: marca usada no HTML para definir elementos.
- **Terminal**: interface por texto para executar comandos.

## U

- **URL**: endereço de um recurso na web.

## V

- **Variável**: espaço nomeado para guardar um valor.
- **Versionamento**: controle das mudanças feitas em um projeto ao longo do tempo.

## W

- **Web**: conjunto de páginas e aplicações acessadas via navegador.
