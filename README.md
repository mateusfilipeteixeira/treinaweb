# Trilha de Estudos: Do Zero à Programação Web

Este repositório agora contém duas coisas ao mesmo tempo:

- a base de conteúdo em Markdown
- um site em Jekyll já preparado para GitHub Pages, domínio próprio, Google Analytics 4 e boas práticas de SEO

O conteúdo foi pensado para pessoas que nunca estudaram TI, nunca programaram e precisam de uma trilha clara, humana e progressiva para chegar com segurança até o desenvolvimento web.

## Para quem este material foi feito

- Para quem está começando do absoluto zero.
- Para quem sente insegurança com termos técnicos.
- Para quem quer aprender com calma, mas sem perder profundidade.
- Para quem quer construir base forte antes de tentar frameworks e ferramentas mais avançadas.

## Como usar esta trilha

1. Estude os módulos na ordem.
2. Assista aos vídeos com papel, bloco de notas ou arquivo de anotações aberto.
3. Reproduza exemplos com as suas próprias mãos.
4. Faça os exercícios antes de olhar respostas prontas.
5. Só avance quando o checklist do módulo fizer sentido para você.
6. Termine cada módulo com o projeto sugerido, mesmo que simples.

## Ritmo sugerido

- Ritmo leve: 5 a 6 horas por semana. Duração estimada de 6 a 8 meses.
- Ritmo equilibrado: 8 a 10 horas por semana. Duração estimada de 4 a 6 meses.
- Ritmo intenso: 12 horas ou mais por semana. Duração estimada de 3 a 4 meses.

O mais importante não é correr. O mais importante é terminar cada etapa entendendo o que está fazendo.

## Estrutura técnica do site

- `index.md`: página inicial do site
- `_config.yml`: configurações do Jekyll, SEO e Analytics
- `_layouts/default.html`: layout principal
- `_data/navigation.yml`: navegação e cards da home
- `assets/css/site.css`: visual do site
- `assets/js/site.js`: menu mobile e consentimento de analytics
- `.github/workflows/deploy.yml`: publicação automática no GitHub Pages

## Publicação

O material está preparado para publicação com Jekyll no GitHub Pages.

Antes de publicar, ajuste no `_config.yml`:

- `url`
- `baseurl`
- `google_analytics`

Depois siga o guia em [Publicação, Domínio e Analytics](./publicacao/index.md).

## Método sugerido para cada módulo

Em todos os módulos, use esta rotina:

1. Entenda o objetivo do módulo.
2. Assista aos vídeos principais.
3. Leia os complementos escritos.
4. Refaça os exemplos sem copiar no automático.
5. Faça os exercícios mínimos.
6. Construa o projeto do módulo.
7. Revise o checklist final.

## O que a pessoa deverá saber ao final

Ao terminar esta trilha, a pessoa deverá ser capaz de:

- Usar o computador e o navegador com mais autonomia.
- Entender conceitos básicos de internet, web e segurança digital.
- Pensar soluções com lógica e algoritmos.
- Criar páginas com HTML semântico.
- Estilizar interfaces com CSS e responsividade básica.
- Programar interações com JavaScript.
- Manipular o DOM, formulários e armazenamento local.
- Versionar projetos com Git e publicá-los no GitHub.
- Entregar pelo menos um projeto web completo e publicado.

## Regras de ouro para quem está começando

- Não compare seu começo com o meio de outra pessoa.
- Errar faz parte do estudo de programação.
- Copiar sem entender não conta como prática.
- Revisão é tão importante quanto conteúdo novo.
- Projeto pequeno concluído vale mais do que projeto enorme abandonado.

## Observação importante

Esta base foi montada para servir como ponto de partida sólido. Ela já é suficiente para orientar meses de estudo, mas pode e deve ser adaptada com o tempo conforme o perfil da turma, o ritmo das pessoas e os objetivos finais de cada uma.
